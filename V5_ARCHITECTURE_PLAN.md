# Enbang.online v5 Architecture Plan — AI Creator Network

## Phase 1 Audit

- **Framework:** Next.js 15 App Router with React 19, TypeScript, Tailwind CSS, Framer Motion, and Lucide icons.
- **Routing:** Locale-scoped routes live under `src/app/[locale]` with English, Chinese, and Japanese support. Root redirects to the default locale. Existing modules include home, lab, tools, assistant, community, knowledge, changelog, global, sitemap, and robots.
- **Components:** Shared glassmorphism UI exists in `src/components`, with reusable shell components in `src/components/ecosystem`.
- **Database:** No existing database client, migrations, or ORM schema were present. v5 introduces PostgreSQL + Prisma without removing static data fallbacks.
- **Authentication:** No existing authentication provider was present. v5 introduces GitHub OAuth-ready route handlers with Google OAuth placeholders and a session helper that can later be swapped for an Auth.js adapter.
- **Deployment:** Deployment is standard Next.js. Environment variables are documented in `.env.example`; build output is generated by `next build`.

## V5 Target Architecture

Enbang.online evolves from a single-person AI Studio into an AI Creator Network while preserving all v4 routes and static content.

### New Product Modules

1. `/creator`
   - Creator discovery and featured profiles.
   - Dynamic `/creator/[username]` profile pages.
   - Profile content: avatar, bio, skills, projects, articles, tools, social links.
2. `/dashboard`
   - Auth-aware user workspace.
   - Management panels for projects, articles, tools, products, and analytics.
3. `/marketplace`
   - AI tools, prompts, and templates.
   - Fields: name, description, price, creator, category, url.
4. `/api`
   - Auth routes via `/api/auth/signin`, `/api/auth/callback/github`, and `/api/auth/signout`.
   - Knowledge Q&A endpoint via `/api/assistant/knowledge`.
   - Analytics event endpoint via `/api/analytics/event`.

## Database Plan

Because no database existed, v5 uses PostgreSQL + Prisma.

Core models:

- `User`
- `CreatorProfile`
- `Project`
- `Article`
- `Tool`
- `Product`
- `Follow`

Additional support models:

- OAuth compatibility: `Account`, `Session`, `VerificationToken`.
- Analytics: `AnalyticsEvent` for page views, clicks, users, projects, and custom network events.

## Authentication Plan

- Primary provider: GitHub OAuth.
- Reserved provider: Google OAuth, activated when `GOOGLE_CLIENT_ID` and `GOOGLE_CLIENT_SECRET` exist.
- Required environment: `AUTH_SECRET`, `AUTH_GITHUB_ID`, `AUTH_GITHUB_SECRET`, `DATABASE_URL`.
- Auth functionality: register/login through OAuth sign-in, session reading, logout through auth route handlers.

## Security Checklist

- Keep secrets in environment variables only.
- Do not expose database URLs or OAuth secrets to the client.
- Restrict dashboard write actions to authenticated users in future server actions.
- Validate API payloads before persistence.
- Keep Prisma queries server-side.

## Rollout Notes

- Existing v4 routes and content remain intact.
- Static seed data powers pages until production database records are migrated.
- Prisma schema is ready for `npx prisma migrate dev` once a PostgreSQL database is available.
